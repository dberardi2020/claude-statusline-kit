# Docs

Documentation for the **Claude Code Statusline Kit**.

- **[product/overview.md](product/overview.md)** — what it is, who it's for, and what each
  segment means.
- **[technical/design.md](technical/design.md)** — the statusline JSON contract, how each
  segment is computed, the self-install mechanism, and how cross-platform parity is kept.
- **[technical/testing.md](technical/testing.md)** — the golden-parity test approach, the
  `SL_NOW` clock seam, the PS 5.1 BOM requirement, and CI.
- **[decisions/](decisions/)** — architecture decision records (ADRs).
- **[tickets/Tickets.md](tickets/Tickets.md)** — the lightweight backlog (open items, no
  external tracker).

The user-facing quickstart lives in the repo [README](../README.md); the styled one-page
reference is [statusline.html](statusline.html).

## Rendering

Prose docs follow an MD-source + HTML-render convention. Most `.html` here is maintained by
hand alongside its `.md`; the tickets page is generated by a small stdlib-only renderer —
after editing `tickets/Tickets.md`, regenerate its render with:

```sh
python docs/render.py docs/tickets/Tickets.md
```
