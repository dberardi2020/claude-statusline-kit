Claude Code · Statusline Kit

Statusline

A two-line Claude Code statusline — model · context · rate-limits, then cwd · branch · cost · elapsed — bracketed by rules. Cross-platform: bash on macOS/Linux, PowerShell on Windows.

two-line layoutreads stdin JSONbash + PowerShell
One statusline, two shells. It surfaces at a glance what Claude Code otherwise leaves implicit: which model you're on, how full the context window is, how close each rate-limit window is to resetting, and the session's branch and cost.

Two implementations

bashmacOS · Linux

statusline-command.sh — self-installing: bash statusline-command.sh --install wires it into ~/.claude/settings.json. macOS bash 3.2-safe; parses stdin with jq.

Verified against 0% / mixed / high-usage payloads, and graceful degradation when rate-limit fields are absent.

PowerShellWindows

statusline.ps1 — the Windows implementation, self-installing via ./statusline.ps1 -Install. PS 5.1-safe emoji via ConvertFromUtf32.

Same JSON schema and layout as the bash version — one statusline in two shells.

Both read the same JSON schema and produce the same layout — one statusline in two shells. Pick the one that matches your platform, or install both if you work across machines.

Segments

#SegmentSource field(s)Format & colour
1🤖 Modelmodel.display_nameBracketed name with the "(1M context)" annotation stripped; white.
2·3Context bar + %context_window.used_percentage10-cell ▓/░ bar + percent. green ≤60 · yellow ≤85 · red >85.
4⏳ 5-hour limitrate_limits.five_hour.resets_at · .used_percentage[Hh Mm] pct% — bracketed countdown (white) then percent (coloured, 60/85).
5📅 7-day limitrate_limits.seven_day.resets_at · .used_percentage[Dd Hh] pct%.
6📁 Directoryworkspace.current_dir / cwdLeaf of the working directory; white.
7🌿 Branch.git/HEAD of the cwd (walked up)Current branch, or --- when not a repo.
8💰 Costcost.total_cost_usdSession cost, $F2.
9⏱️ Elapsedcost.total_duration_msWall-clock → Hh Mm (drops the hour under 1h).

Line 1 = model · context bar · context % · the two rate-limit windows. Line 2 = the fixed context row (dir · branch · cost · elapsed). The layout is two pipe-delimited lines wrapped in ─×71 rules.

Shared behaviour

  • Safe field access: every value read through a guarded helper (Get-Safe in PS, is_set + jq // defaults in bash) — a missing key degrades the line instead of erroring.
  • Graceful segments: when the rate-limit fields are absent, those segments drop out and line 1 is just model + context (verified in test).
  • Colour policy: green ≤60 / yellow ≤85 / red above, applied to the context bar and each rate-limit percent; countdowns and labels stay white.
  • Rate-limit countdowns: resets_at (unix seconds) minus now, floored at 0, split into d/h/m.

Install

Each script is self-installing — one download, one command. It copies itself into ~/.claude/ and merges a statusLine entry into ~/.claude/settings.json, backing the file up first and preserving your other settings.

  • macOS / Linux: bash statusline-command.sh --install (requires jq).
  • Windows: ./statusline.ps1 -Install.

See README.md for the one-line download commands, a copy-paste prompt to hand your coding agent, and the manual settings.json snippets.
